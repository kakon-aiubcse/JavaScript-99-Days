JavaScript Advanced Concepts & Interview Questions

  1. What is the difference between == and === in JavaScript?


    Answer:


     == (Abstract Equality): Compares values after type conversion (loose comparison).

     === (Strict Equality): Compares values without type conversion (strict comparison).


    js CopyEdit

    console.log(5 == "5"); // true (type conversion happens) console.log(5 === "5"); // false (different types)


  2. What are closures in JavaScript?


    Answer:

    A closure is a function that retains access to its lexical scope even when the function is executed outside of its scope.


    js CopyEdit

    function outer() { let count = 0;

    return function inner() { count++; console.log(count);

    };

    }

    const counter = outer(); counter(); // 1

    counter(); // 2


  3. What is the event loop in JavaScript?


    Answer:

    The event loop handles asynchronous operations. It continuously checks the call stack and the task queue to execute pending tasks.


    js CopyEdit

    console.log("Start");

    setTimeout(() => console.log("Timeout"), 0); console.log("End");

    // Output: Start → End → Timeout


  4. What is hoisting in JavaScript?


    Answer:

    Hoisting moves function and variable declarations to the top of their scope before execution.

    js CopyEdit

    console.log(a); // undefined (due to hoisting) var a = 5;


  5. Explain the difference between var, let, and const.


    Answer:


    • var: Function-scoped, can be redeclared.

    • let: Block-scoped, cannot be redeclared.

    • const: Block-scoped, immutable reference.


      js CopyEdit

      var x = 10; // Function scope let y = 20; // Block scope

      const z = 30; // Block scope, cannot be reassigned


  6. What is a promise in JavaScript?


    Answer:

    A promise is an object that represents the eventual completion or failure of an asynchronous operation.


    js CopyEdit

    const myPromise = new Promise((resolve, reject) => { setTimeout(() => resolve("Success"), 1000);

    });

    myPromise.then(console.log); // "Success" after 1 sec


  7. What are async/await in JavaScript?


    Answer:

    Async/await allows writing asynchronous code in a synchronous style.


    js CopyEdit

    async function fetchData() {

    let data = await fetch("https://api.example.com"); console.log(data);

    }


  8. What is the difference between null and undefined?


    Answer:


    • null: Intentional absence of value.

    • undefined: Variable declared but not assigned a value.

      js CopyEdit let a;

      console.log(a); // undefined let b = null; console.log(b); // null


  9. What are arrow functions, and how do they differ from regular functions?


    Answer:

    Arrow functions do not have their own this, they inherit this from the surrounding scope.


    js CopyEdit

    const add = (a, b) => a + b; console.log(add(2, 3)); // 5


  10. What is the difference between synchronous and asynchronous programming?


    Answer:


    • Synchronous: Code executes line by line, blocking execution.

    • Asynchronous: Code executes non-blocking, allowing other tasks to run.



      Common JavaScript Interview Questions

  11. How does JavaScript handle memory management?


    JavaScript uses garbage collection, automatically removing unreferenced objects from memory.


  12. What are prototypes in JavaScript?


    Every JavaScript object has an internal prototype (  proto  ) that allows inheritance.


    js CopyEdit

    function Person(name) { this.name = name;

    }

    Person.prototype.greet = function() { return "Hello, " + this.name;

    };


  13. What are template literals in JavaScript?


    Template literals use backticks (`) for string interpolation.

    js CopyEdit

    let name = "Alice";

    console.log(`Hello, ${name}!`); // Hello, Alice!


  14. What is destructuring in JavaScript?


    Destructuring extracts values from arrays or objects easily.


    js CopyEdit

    let [a, b] = [1, 2];

    let { x, y } = { x: 10, y: 20 };


  15. Explain the difference between map, filter, and reduce.


    • map(): Transforms each array element.

    • filter(): Returns elements that meet a condition.

    • reduce(): Accumulates values into a single result.


  16. What is this in JavaScript?


    The this keyword refers to the current execution context.


  17. What is debouncing and throttling?


    • Debouncing: Delays execution until no further calls.

    • Throttling: Limits execution to once per interval.


  18. What is deep vs shallow copy?


    • Shallow copy: Copies only references.

    • Deep copy: Copies all nested objects.


  19. What is event delegation?


    Handling events at a parent level instead of each child element.


  20. What is the difference between setTimeout and setInterval?


    • setTimeout(): Executes once after delay.

    • setInterval(): Executes repeatedly at intervals.


  21. How does JavaScript handle concurrency?

    JavaScript uses a single-threaded, non-blocking event loop to handle concurrency. It processes tasks asynchronously using callbacks, promises, and async/await, avoiding blocking operations.



  22. What is the difference between function declaration and function expression?

    • Function Declaration: Hoisted to the top of the scope.

    • Function Expression: Not hoisted, assigned to a variable.


      js CopyEdit

      // Function Declaration function greet() {

      return "Hello";

      }


      // Function Expression const greet = function() {

      return "Hello";

      };



  23. How do you deep clone an object in JavaScript?

    A deep clone creates a copy of an object, including nested objects.


    js CopyEdit

    const obj = { a: 1, b: { c: 2 } };

    const clone = JSON.parse(JSON.stringify(obj));


    Alternatively, use structuredClone() in modern browsers:


    js CopyEdit

    const deepCopy = structuredClone(obj);



  24. What is call, apply, and bind?

    • call(): Calls a function with arguments passed separately.

    • apply(): Calls a function with arguments as an array.

    • bind(): Returns a new function with bound this.


      js CopyEdit

      function greet(greeting) { console.log(`${greeting}, ${this.name}`);

      }

      const person = { name: "Alice" };


      greet.call(person, "Hello"); // Hello, Alice greet.apply(person, ["Hi"]); // Hi, Alice const boundGreet = greet.bind(person); boundGreet("Hey"); // Hey, Alice



  25. Explain the difference between synchronous and asynchronous iterators.

    • Synchronous Iterators: Return values immediately.

    • Asynchronous Iterators: Return a promise, useful for streaming data.


      js CopyEdit

      async function* asyncGen() { yield Promise.resolve(1); yield Promise.resolve(2);

      }


      (async () => {

      for await (let num of asyncGen()) { console.log(num);

      }

      })();



  26. What is an Immediately Invoked Function Expression (IIFE)?

    An IIFE is a function that runs immediately after being defined.


    js CopyEdit

    (function () {

    console.log("I run immediately!");

    })();



  27. How does Object.freeze() work?


    Object.freeze() makes an object immutable, preventing property changes.


    js CopyEdit

    const obj = { name: "Alice" }; Object.freeze(obj);

    obj.name = "Bob"; // No effect



  28. What are JavaScript generators?

    Generators allow pausing and resuming execution.


    js CopyEdit

    function* gen() { yield 1;

    yield 2;

    }

    const g = gen(); console.log(g.next().value); // 1

    console.log(g.next().value); // 2



  29. Explain WeakMap and WeakSet.


    • WeakMap: Stores key-value pairs where keys are weakly referenced.

    • WeakSet: Stores unique values that are objects.


      js CopyEdit

      let weakMap = new WeakMap(); let obj = {}; weakMap.set(obj, "data");

      console.log(weakMap.get(obj)); // "data"



  30. What is the difference between push() and concat()?

    • push(): Modifies the original array.

    • concat(): Returns a new array.


      js CopyEdit

      let arr = [1, 2];

      arr.push(3); // [1, 2, 3]

      let newArr = arr.concat([4, 5]); // [1, 2, 3, 4, 5]

    Modules allow code to be split into reusable files.
